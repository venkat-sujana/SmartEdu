# OSRA - Open Source Record Assistant

Version: 1.0

# Purpose

This repository contains a production-grade ERP platform for Government Junior Colleges.

The AI assistant must understand the existing architecture before making any code changes.

Never rewrite the application unnecessarily.

Always preserve existing functionality.

# AI Goal

The AI assistant should:

- understand existing code
- reuse components
- preserve architecture
- avoid duplicate logic
- produce production-ready code

---

# Editing Rules

Before modifying code:

Read existing implementation.

Understand dependencies.

Identify reusable components.

Avoid unnecessary refactoring.

Modify only required files.

Never rename folders unless requested.

Never break APIs.

Never change database schema without instruction.

---

# General Coding Rules

Always prefer

- readability
- maintainability
- modularity
- simplicity

Avoid

- duplicated logic
- giant components
- magic numbers
- unnecessary libraries

---

# Response Rules

When generating code

Always

- return complete updated file
- explain major changes briefly
- preserve formatting
- keep imports clean

Never generate placeholder code.

Never remove working features.
